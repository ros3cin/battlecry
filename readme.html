1. Introduction

1.1 What is Battlecry?

First of all, don't get me wrong: I really like heavy metal music. Among the countless bands, there are some around that have rather... ahem... interesting lyrics. Did you ever want to write songs like that, but you just coudn't think of the right lyrics? No problem with "Battlecry"!
Battlecry loads a data file that contains word lists, grammatical structures and layout information and generates a random set of lyrics according to that information. It was designed with heavy metal lyrics in mind, but modules can be written for any other type of song.
This tool's intention is to be entertaining; however, if you really want to use its output as lyrics for a real song, you are free to do so. In that case, please send me a demo tape... :-)

2. How To Use

2.1 Installation

Battlecry is written in Java, so in order to run it (or any other Java application), you need the Java 2 Runtime Environment. If you want to compile the program yourself, you need the Java 2 SDK. Both are available for free download at java.sun.com. Battlecry should run on any platform that can run Java applications; it was tested on Linux and Windows.
Once you have it installed, unzip the battlecry archive to your hard drive. Start the program by typing "java battlecry" at the command line prompt. If no module file is given (with the -m switch), the program will attempt to load the standard module at data/battlecry.pak. If you want to use the graphical user interface (GUI), type "java frontend" instead.

2.2 How to use

At the moment, Battlecry only loads a module, generates a set of lyrics, displays it on the screen and exits. To change something, you need to edit a module or create a new one. See section 3 for details.

If you use the command line interface ("java battlecry") you can use the parameter -h to get a list of options.

Alternatively, you can use the graphical user interface ("java frontend"). You can choose a module file to load, generate lyrics and use the layout field on the upper right of the screen to use a custom layout (see section 3.2 for details). After typing the layout you must hit the reload button for the changes to take effect.

3. Battlecry Modules

3.1 What is a Battlecry Module?

Battlecry gets its information from a single data file, a Battlecry Module. This is actually a plain zip file containing grammar and layout structures as well as word lists. The "layout.dat" file holds information about the outer form - verses, choruses, rhyme schemes ("aabb", "ababcc",...) and number of lines. The "grammar.dat" file states what a single line of text can look like (example: "We [verb] the [verb] of the [noun]."). Verbs, nouns, adjectives etc. are randomly loaded from the specific word list (in this example, the files "noun.lst" and "verb.lst" must exist inside the module). Finally, every module has a file called "info.dat" which contains name, author, version and description of a module. A detailed description follows.

3.2 The Layout File

Every Battlecry module needs a file called "layout.dat". At the moment, there are four different keywords that can be used: "lines", "verse", "bridge", and "chorus". Every keyword is followed by a ":" and a number / rhyme scheme. The keywords do the following:
-"lines:n" generates a bunch of completely random, non-rhyming lines where n is the number of lines.
-"bridge:n" generates n random lines, every second line rhymes with its predecessor. If n is an odd number, it will be automatically increased by 1.
-"chorus:s" generates rhyming lines according to the given rhyme scheme s. s is a string  which constists of the characters a,b,c,d,e, each representing a rhyming line. Each character must not appear more than three times!
-"verse:s" generates rhyming lines according to the rhyme scheme s.  Unlike the chorus, verses are not repeated but newly generated  every time.
Here is an example layout.dat file:

verse:aabbccdd
chorus:aabbcc
lines:6
chorus:aabbcc
bridge:2
chorus:aabbcc

This will generate 4 pairs of rhyming lines, followed by a chorus of 3 pairs of rhyming lines, then 6 random lines, repeated chorus, 2 rhyming lines, repeated chorus.
Note that verses and bridges are different everytime, while the chorus is only generated once and repeated; thus, the rhyme scheme is interpreted only in the first "chorus" line.
Apart from those keywords, you can use the names of segments from "grammar.dat" - see below for details.
Hint: The command line parameter "-l <filename>" allows you to use another layout file instead of the one inside the module.
If you use the GUI, you can type a valid layout right into the layout field and hit the "Reload" button.

3.3 The Grammar File

Every Battlecry module must contain a file called "grammar.dat". It holds information about how a line of text is formed. Let's look at a simple example:

START={[EXAMPLE1]#[EXAMPLE2]}
TITLE={[TITLE1]#[TITLE2]}

TITLE1={noun;'of';*'the';noun}
TITLE2={verb;'for'#'to';*'the';noun}

EXAMPLE1={'We';verb;'to';verb;*'the';noun}
EXAMPLE2={'We fight for the';noun;*[OFTHE]}
OFTHE={'of';*'the';noun}

The capital words are segments (they don't need to be capital but it's easier to recognize). Each segment looks like this: SEGMENTNAME=(expression).
Expressions are built from one or more of the following parts, separated by a semicolon (;):
    -Strings: Actual words that appear in that line of text, marked by '. Example: 'We fight for the'
    -Words from a list: Unmarked Strings are interpreted as names of list files. Example: noun will be replaced by a random word from the file "noun.lst"
    -Segments: The name of another segment, marked by []. Example: noun;[OFTHE] will be interpreted as noun;'of';*'the';noun.

Modifiers (can be used before strings, word lists and segments):
    * marks a part as optional. The randomizer decides if that part is used in a line or not. Example: 'Sword of';*'the';'King' can turn out as "Sword of the King" or "Sword of King".
    # means "or". Example: [TITLE1]#[TITLE2]#[TITLE3] randomly chooses either TITLE1 or TITLE2 or TITLE3.

Update: Since 0.62, Battlecry can handle brackets inside grammatical segments, for example:
START={'Fight';('for the';noun)#('to';verb)}
Brackets inside brackets are not allowed yet.

Important: The "grammar.dat" file must contain at least two lines: START={expression} and TITLE={expression}, where START is the beginning of every line and TITLE is the title of the song.

3.4 The Word Lists

The number and names of the needed word lists depend on the grammatical structure described in the "grammar.dat" file. For the above example, two word lists are needed: "noun.lst" and "verb.lst". Here is an example word list:

IFE;life,knife,wife
AR;star,car
ARROT;carrot

The capital part is the rhyming syllable, followed by a semicolon and one or more words, separated by commas. Battlecry interprets all words as rhyming that have exactly the same rhyming syllable. This is important so words from different lists are found rhyming. For example, to make sure "arrive" rhymes with "life", add the line "IFE;arrive" to the "verb.lst" file.
Note: Instead of IFE/AR/ARROT, 1/2/3 would work as well, but that'll get very confusing with big word lists.
Hint: If you use segments in "grammar.dat" that end with a fixed string (example: TITLE=('Let us';verb;'until tonight')), make sure the last word ("tonight") appears in a word list so that line can rhyme with another line. If it doesn't fit in any of the lists, create a new one for those "last words". Battlecry will load it although it isn't used by the grammar.

3.5 The Info File

The file info.dat structure is very straightforward. It holds a few module options and information about the module (i.e. name, author, version and comment) that can be accessed by using the "-i <filename> command line option. Here's an example:

NAME=Example Module
AUTHOR=Some Random Guy
VERSION=0.1
COMMENT=This is an example module.
OPTION_SYLLABLE_TOLERANCE=3

OPTION_SYLLABLE_TOLERANCE is the amount of syllables in that two or three rhyming lines may differ. A lower value may result in longer loading time while a higher value will cause lines that differ in length.
Experiment with this option to get the best results out of your module.

3.6 Editing And Creating Modules

To edit a module, simply unzip it, edit the .dat or .lst files, put them back into an archive and load it with Battlecry.
When creating new modules, be careful to use correct syntax, especially in the .dat files. Don't create infinite loops in your segment structure, and don't use directories inside the zip archive. Since Battlecry is still in early stages it doesn't yet handle all exceptions.
